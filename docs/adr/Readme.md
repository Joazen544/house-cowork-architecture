# ADR 指引

## ADR 檔案命名規則

每個 ADR 檔案應該依照以下格式命名：

```
<序號>-<Status縮寫>-<標題>.md
```

例如：`0001-AC-adr-tools-selection.md`

### 常見的 Status 及其縮寫

- **Proposed (PR)**: 提議中
- **Accepted (AC)**: 已接受
- **Rejected (RJ)**: 已拒絕
- **Deprecated (DP)**: 已棄用
- **Superseded (SP)**: 已取代

## ADR 模板

每個 ADR 應該包含以下部分：

### 1. 標題

```
# <序號>-<Status縮寫>-<標題>
```

### 2. Status

```
## Status

<Status>
```

### 3. Context

```
## Context

<描述決策背景和原因>
```

### 4. Decision Drivers

```
## Decision Drivers

<描述影響決策的因素>
```

### 5. Decision

```
## Decision

<描述做出的決策>
```

### 6. Alternatives Considered

```
## Alternatives Considered

<描述考慮過的替代方案>
```

### 7. Consequences

```
## Consequences

<描述決
```
